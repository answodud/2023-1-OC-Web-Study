1-1. add
git add는 작업 디렉토리 상의 변경 내용을 스테이징 영역에 추가하기 위해 사용하는 명령어이다. 다음 변경(commit)을 기록할 때까지 변경분을 모아놓기 위해 사용한다.
즉. git commit을 통해 명시적으로 기록을 남기기 전까지는 아무리 git add 명령어를 많이 실행한다 해도 저장소의 변경 이력에는 영향이 없다.

git add <파일> //작업 디렉토리의 변경 내용의 일부만 스테이징 영역에 넘기고 싶을 때

git add . //현재 디렉토리의 모든 변경 내용을 스테이징 영역으로 넘기고 싶을 때

git add -A //작업 디렉토리 내의 모든 변경 내용을 다 스테이지 영역으로 넘기고 싶을 때(git add . 을 프로젝트 최상위 디렉토리에서 실행할 경우 동일한 효과임)

git add -p //각 변경 사항을 터미널에서 직접 눈으로 하나씩 확인하면서 스테이징 영역으로 넘기거나 제외할 수 있음. 

1-2. commit
commit은 변경사항을 내 컴퓨터에 저장하는 역할을 한다. 의미 있는 변화에 대해 기록하는 것.

git add <파일> 
git commit -m "<메시지>" 수정 
// 파일 스테이징 하고 커밋하기

git commit -m "<메세지>" -a //수정되고 추적되는 모든 파일의 변경사항 커밋하기

git commit -m "<메세지>"--amend // 마지막 커밋 고치기

git commit -C HEAD--amend // 이전 커밋을 수정하고 커밋 메세지를 재사용하기

1-3. push
push 명령어는 현재 프로젝트의 커밋된(HEAD) 내용을 원격 저장소로 내보내는 명령어이다. push로 내보내게 되면 원격 저장소에 저장되게 되며 이러한 저장된 내용은 다른 사용자들도 이용할 수 있게 된다.

git push <원격 저장소명><branch 이름>

1-4. fetch와 pull
git fetch는 원격 저장소에 있는 변경 사항들을 로컬 저장소에 가져오기 전에 변경 내용을 확인하고 싶은 경우에 사용하는 명령어이다. 즉, 병합을 하기 전 병합을 할지 말지 확인 할 수 있는 명령어이다.

git fetch 원격저장소 이름
git pull은 원격 저장소에 있는 변경 사항들을 로컬 저장소로 가져와 합치는 명령어이다. fetch랑 다르게 로컬 디렉토리에 변경 내용을 병합한다.

둘 다 원격 저장소의 내용을 가져오는데 사용된다. fetch는 변경 내용을 가져오지만 로컬에 영향을 미치지 않으며, 병합 전 확인 용도로 쓴다. pull은 가져온 변경 내용을 로컬에 병합한다. 로컬에서 작업하다가 변경된 내용을 pull할 경우 충돌이 일어날 수 있기에 fetch 후 pull을 사용하는 것이 좋다.
